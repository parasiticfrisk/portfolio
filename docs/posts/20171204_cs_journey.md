When I first enrolled at MJC, my goal was simple: earn a technical certification in computer networking. I wanted to build practical skills that would help me start a career in IT. What I did not expect was that an introductory programming course would completely change the direction of my education.

The end of this semester marked the completeion of Intro to Programming and the beginning of my computer science journey. It also helped me to realize how much I enjoy creating software and the creative outlet for solving problems that writing code provides.

## Final project: `personality_calculator`
For my final project, I developed [personality_calculator], a desktop application written in [Python] 3.6 using only the [standard library] and the built-in [IDLE] development environment. The program uses a [graphical user interface] to collect information from the user and generate several personality related results based on their date of birth. The application calculates [personality type], [western zodiac] sign, [chinese zodiac] sign, and [biorhythm] information.

:::{important}
The project was not intended to lend legitimacy to any of these [pseudosciences].
:::

After the user closes the GUI, the program automatically launches their default web browser and displays the generated results. To add [persistence] and [data management], the application also stores user activity in a [log file] and saves user input and generated results in an [SQLite] database.

:::{seealso}
For more technical information about the project, see [Building a portable desktop application using only the Python standard library].
:::

## Learning beyond the classroom
The greatest influence on my experience was Professor [Dale Phillips]. His passion and enthusiasm for computer systems, programming, and exploration is contagious. He consistently encouraged students to explore beyond the course materials and challenge what we thought we were capable of achieving. His advice was always practical and informed by years of expereince.

:::{hint} Some of Dale's recurring advice
:class: simple
* Learn [Linux] and [operating systems]
* Use a [version control system]
* Learn [SQL]
* Explore [web development]
* Check out [WebAssembly]
* Learn [computer networking]
* Do not be afraid to break things!
:::

While all of those lessons were valuable, one idea stood out to me more than the others: **write clean code**.

Professor Phillips emphasized creating software using a [self-documenting] architeture. Instead of relying on extensive [comments] to explain confusing code, the goal is to write software that clearly communicates its purpose through structure, naming, and organization. It is a lesson I expect to carry with me throughout my career.

Another practical recommendation was creating a [GitHub] account and learning to use version control. At the time, it seemed like a small suggestion. Looking back, it has become one of the most valuable tools in my learning journey and is the reason I am able to share this project and experience today.

## The value of teamwork
Although `personality_calculator` was an individual project, I would not have had the same experience without my study partners [Harpreet Hothi] and [Vanessa Davis]. We spent countless hours working through assignments, discussing programming concepts, and exploring material beyond what was required for the course. Through these study sessions, I learned the value of collaboration in software development.

[Pair programming], [scaffolded learning], and simply having a team of people willing to tackle difficult problems together made a tremendous difference. Some of the most important lessons came from explaining concepts to one and working through challenges as a group. Working with Harpreet and Vanessa helped solidify my decision to pursue a degree in Computer Science. They showed me that software development is as much about collaboration, communication, and continuous learning as it is about writing code.

## Looking forward
When I started college, I expected to leave with computer networking skills. Instead, I discovered a passion for software development. Building `personality_calculator` taught me more than Python [syntax]. It taught me how to approach problems, design software, manage data, present technical work, and collaborate with others. The project represents my first complete software application and my first opportunity to present a program that I built from the ground up. While there are certainly things I would do differently, it stands as an important milestone in my growth as a developer.

The first chapter of this journey is complete. Now it is time to see where the next one leads.
